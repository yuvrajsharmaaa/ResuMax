## ğŸ“š NLP Concepts & Theory

### 1. **Text Preprocessing**

#### Tokenization
Breaking down text into individual words or tokens.

```
Input: "Experienced software developer with Python skills"
Output: ["Experienced", "software", "developer", "with", "Python", "skills"]
```

**Why it matters:** Tokenization is the foundation for all subsequent NLP tasks. It allows the system to analyze individual components of the resume.

#### Stop Word Removal
Removing common words that don't add semantic value.

```
Before: ["I", "am", "a", "skilled", "developer", "with", "expertise"]
After: ["skilled", "developer", "expertise"]
```

**Why it matters:** Focuses analysis on meaningful content words, improving efficiency and accuracy.

#### Lemmatization
Converting words to their base/dictionary form.

```
"developing" â†’ "develop"
"experiences" â†’ "experience"
"managed" â†’ "manage"
```

**Why it matters:** Ensures different forms of the same word are treated identically, improving matching accuracy.

### 3. **Keyword Extraction & TF-IDF**

**Term Frequency-Inverse Document Frequency (TF-IDF)** identifies important keywords.

#### Formula:
```
TF-IDF(t, d) = TF(t, d) Ã— IDF(t)

Where:
TF(t, d) = (Number of times term t appears in document d) / (Total terms in d)
IDF(t) = log(Total documents / Documents containing term t)
```

#### Example:
```
Resume contains: "Python" (5 times), "JavaScript" (2 times), "and" (20 times)

TF-IDF scores:
â€¢ "Python" â†’ High (frequent in resume, rare across all resumes)
â€¢ "JavaScript" â†’ Medium
â€¢ "and" â†’ Low (common word, appears everywhere)
```

**Application:** Identifies the most relevant skills and experiences in the resume.
